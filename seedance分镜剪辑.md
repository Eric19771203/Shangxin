## 一、核心定义与本质区别（结合Seedance 2.0官方功能）
### 基础定义
- **首尾帧**：是关键帧的**特殊子集**，仅包含2帧，固定在视频的**第0帧（首帧，视频开头）**和**最后1帧（尾帧，视频结尾）**，核心是锁定视频的起点与终点画面，由AI自动补全两点之间的完整过渡过程。
- **关键帧**：是视频时间轴上**任意位置、定义画面核心变化节点的帧**，数量≥2，不局限于首尾，可在视频的任意时间点设置，每一个关键帧都锁定对应时间点的角色姿态、场景构图、画面元素，AI按时间顺序补全相邻关键帧之间的过渡，是控制视频动作节奏、多节点叙事的核心手段。

### 核心区别对照表（漫剧+Seedance 2.0场景专用）
| 对比维度 | 首尾帧 | 关键帧（多关键帧） |
| :--- | :--- | :--- |
| 帧数量 | 固定2帧（首+尾） | ≥2帧，无上限（Seedance 2.0最多支持12个参考素材，含其他参考图，建议15秒单元不超过4个关键帧） |
| 时间位置 | 固定在视频的起点和终点，不可调整 | 可在视频时间轴的任意位置设置（如15秒视频的0s、5s、10s、15s），可自由定义时间节点 |
| 控制逻辑 | 仅锁定「起点→终点」的动作闭环，AI自由发挥中间过程 | 按时间顺序锁定**每一个核心变化节点**，AI仅能在相邻两个关键帧之间做过渡补全，全程可控性远高于首尾帧 |

## 五、Seedance 2.0关键帧/首尾帧专属避坑指南
1.  **素材数量限制**：Seedance 2.0最多支持上传12个参考素材，多关键帧模式下，关键帧数量+全局参考图+其他素材，总数不得超过12个，15秒单元建议关键帧不超过4个，避免超限。
2.  **关键帧时间戳规范**：必须按**时间正序**绑定关键帧，不可出现时间倒序，提示词中必须明确「@关键帧X 第X秒」，避免模型识别混乱。
3.  **关键帧画面一致性**：同一单元的多个关键帧，必须保证角色、场景、画风完全统一，仅动作、景别、构图发生变化，否则会出现画面跳变、角色崩坏。
4.  **首尾帧动作逻辑**：首尾帧模式下，首帧和尾帧的动作必须有明确的逻辑关联，避免出现“首帧角色在室内，尾帧角色在室外”的跨场景跳转，否则会生成逻辑混乱的画面。
5.  **长视频连贯技巧**：前一个15秒单元的尾帧，直接作为后一个单元的首帧，可大幅提升整集漫剧的镜头连贯性，避免出现切镜跳变。

对，**这个「分镜语义智能分析+自动匹配生成模式」的功能，是你这套系统的核心决策引擎，也是实现「单集漫剧多模式混合、无需人工干预、精准适配Seedance 2.0」的核心前提**。

简单说：没有这个功能，你就只能人工给每个15秒分镜手动选生成方式，达不到「自动判断、自动化生成」的系统级目标；有了这个功能，你只需要输入完整剧本/分镜描述，系统就能自动完成「拆单元→分析分镜→匹配最优生成模式→输出Seedance可直接用的生成指令」全流程，同时彻底避免全程只用一种模式的问题。

## 二、分镜分析功能的核心实现步骤（可直接落地开发）
### 步骤1：分镜文本的结构化拆解（语义提取层，基础核心）
这一步的目标，是把用户写的自然语言分镜描述，拆解成系统可识别的**8个核心量化字段**，所有后续的模式匹配，都基于这8个字段，完全避免主观判断。

#### 必须拆解的8个核心字段（可量化、无歧义）
| 字段名称 | 字段说明 | 拆解示例 |
| :--- | :--- | :--- |
| 单元基础信息 | 固定15秒时长、单元序号、时间区间 | 单元4，45-60s，15秒 |
| 叙事功能 | 固定选项：开场定调/人物特写/对话推进/动作衔接/冲突高潮/蒙太奇快剪/氛围渲染/结尾留钩 | 冲突高潮/打斗戏 |
| 动作节点数量 | 15秒内，独立的、有明确起止的动作数量（核心判断字段） | 4个（挥拳→侧身躲开→抓手腕→按墙上） |
| 画面变化节点 | 15秒内，景别/运镜/场景/角色情绪的变化次数 | 3个（中景→近景→特写，情绪从平静→凶狠） |
| 角色数量 | 单镜头内出现的角色个数 | 2个（男主+反派） |
| 运镜复杂度 | 固定选项：固定机位/单一运镜/多节点运镜 | 多节点运镜（跟拍+推进+切换景别） |
| 画面控制强度 | 固定选项：低（仅需还原画面）/中（需锁定动作起点终点）/高（需精准控制每个动作节点） | 高 |
| 漫画风格诉求 | 固定选项：无/强漫画分格感/强氛围质感 | 无 |

#### 拆解实现方式（两种可选，适配你的技术能力）
1.  **轻量版（无代码，快速落地）**：给用户做标准化的分镜录入模板，用下拉菜单固定选项，用户直接勾选/填写核心字段，无需系统做语义解析，适合Excel/飞书多维表格搭建的简易系统
2.  **完整版（自动化，系统级）**：接入大语言模型（豆包大模型/GPT-4o-mini，优先选豆包，和Seedance同生态适配性更强），写固定的prompt让AI自动拆解分镜文本，输出结构化的8个字段，无需人工填写。

##### 完整版拆解专用prompt（直接复制可用）
```
你是专业的漫剧分镜解析助手，现在需要将用户输入的分镜描述，拆解为8个固定的结构化字段，严格按照要求输出，不添加额外内容。
要求：
1.  严格按照给定的字段选项输出，不得自定义选项
2.  动作节点数量：只统计有明确起止的独立动作，微表情/静态站位不计入
3.  画面变化节点：只统计景别、运镜、场景、角色核心情绪的变化，无明确变化不计入
4.  输出格式：纯文本，每行一个字段，格式为【字段名】：字段值
```

### 步骤2：量化打分与模式初匹配（核心决策层）
基于拆解好的8个字段，给6种生成模式分别打分（满分100分），**单单元得分最高的模式，为初定生成模式**。

#### 核心打分规则（和之前的模式定位完全对齐，可直接写入系统）

#### 强制触发规则（优先级高于打分，直接锁定模式）
为了避免打分偏差，设置3条不可突破的强制规则，直接触发对应模式：
1.  动作节点数量≥3个 → 强制触发C2多关键帧模式
2.  叙事功能为蒙太奇快剪/强漫画分格感 → 强制触发D九宫格分镜模式
3.  叙事功能为开场定调/氛围渲染/结尾留钩，且动作节点为0 → 强制触发E九宫格+镜头描述模式

### 步骤3：整集全局规则校验（防重复、保节奏的核心）
这一步是解决你「不想整个漫剧都用一种方式」的关键，单单元的最优模式，不一定符合整集的节奏和模式多样性要求，所以必须做整集层面的校验与修正，规则如下（优先级从高到低）：

1.  **连续禁用规则**：当前单元的初定模式，不得与前一单元的最终模式一致，若一致，自动顺延至本单元第二高分的模式
2.  **高频限用规则**：单集内同一种模式的使用次数不得超过3次；C2多关键帧模式单集使用不得超过2次，避免素材超限、生成翻车
3.  **全量覆盖规则**：整集8个单元，必须覆盖≥4种模式，且C1/C2模式至少使用1次，若未达标，自动将整集内得分最低的2个单元，替换为未使用模式中适配度最高的模式
4.  **节奏适配规则**：整集的模式排布，必须符合漫剧的叙事节奏（开场→铺垫→冲突→高潮→收尾），不得出现高潮段用A模式、铺垫段用C2模式的错位情况

## 三、两种可直接落地的实现方案
### 方案1：轻量无代码版（1小时就能搭好，适合快速验证）
用Excel/飞书多维表格搭建，核心是「标准化录入+公式自动匹配」，无需写代码，适合个人创作者快速使用：
1.  做2个工作表：【分镜录入表】【模式匹配规则表】
2.  【分镜录入表】设置固定列，用下拉菜单锁定可选值，避免输入不规范
3.  用`IFS`/`VLOOKUP`函数，把录入的字段和规则表匹配，自动输出对应模式
4.  用`CONCAT`函数，自动拼接对应模式的Seedance提示词模板，直接复制可用

### 方案2：系统级自动化版（适合做专属工具，全流程无人干预）
技术架构极简，核心是「大模型语义解析+规则引擎+Seedance适配」：
1.  前端：分镜录入界面，支持粘贴整集剧本，自动拆分15秒单元
2.  中间层：
    - 语义解析模块：接入豆包大模型API，用固定prompt拆解分镜结构化字段
    - 规则引擎模块：写入打分规则、强制规则、全局校验规则，自动匹配生成模式
    - 提示词生成模块：对应模式的模板库，自动拼接输出标准化提示词
3.  输出层：支持导出整集分镜生成包，可批量导入Seedance 2.0，支持一键复制单条指令

## 四、核心避坑与优化建议
1.  **保留人工干预入口**：自动匹配完成后，支持用户手动修改单个单元的生成模式，避免特殊分镜的匹配偏差
2.  **分镜描述越具体，分析越精准**：引导用户在分镜描述中写清动作、景别、运镜、情绪，不要用模糊的抽象词
3.  **适配Seedance素材上限**：多关键帧模式下，自动校验素材总数不超过12个（Seedance参考素材上限），避免超限报错
4.  **增加反馈闭环**：支持用户录入生成效果（是否穿模、动作是否符合预期），系统自动优化打分规则，越用越准
5.  **全局画风锁死**：所有模式的输出指令，都自动追加统一的全局画风约束词，避免不同模式生成的画面割裂
