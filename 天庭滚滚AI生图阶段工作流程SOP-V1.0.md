# 《天庭滚滚》AI生图阶段工作流程SOP
## V1.0 正式版

---

### 一、生图标准规范
#### 1. 基础要求
- **分辨率**：8K（7680×4320）
- **风格**：Q版3D卡通，萌系治愈，色彩明亮
- **核心元素**：每帧必须包含至少3个"圆滚滚"视觉元素
- **人物特征**：所有角色严格遵循30个角色设定标准
- **场景规范**：所有场景严格遵循世界观设定手册

#### 2. 提示词格式标准
```
@[角色设定图] 作为主角, [角色特征], [动作/表情], 背景是[场景描述], [运镜要求], [风格参数], 参考@[场景设定图]
```

#### 3. 风格参数强制要求
- 必须包含：`Cinematic,8K,浅景深,电影感强烈`
- 禁止包含：真人、写实、恐怖、暴力等违规内容

---

### 二、质量校验规则
#### 1. 自动化校验（必过）
- [ ] 分辨率检查：≥7680×4320
- [ ] 人物一致性校验：与设定图相似度≥95%
- [ ] 视觉锤检查：每帧≥3个"圆滚滚"元素
- [ ] 违规内容检测：无违规元素
- [ ] 色彩校验：整体亮度≥60，饱和度适中

#### 2. 人工校验（必过）
- [ ] 剧情连贯性：与分镜描述完全一致
- [ ] 动作合理性：动作自然，符合角色性格
- [ ] 表情准确性：表情与情绪要求匹配
- [ ] 场景完整性：场景元素无缺失
- [ ] 整体美感：符合萌系治愈风格定位

---

### 三、批量生成流程
#### 1. 前置准备
1. 确认所有角色设定图、场景参考图已上传至素材库
2. 确认所有分镜提示词已提取并整理为批量格式
3. 确认生图工具参数配置完成
4. 确认输出目录结构已创建

#### 2. 批量生成步骤
1. **测试批次**：每集先生成前3个单元（0-45秒）作为测试
2. **测试校验**：完成测试批次质量校验，通过率100%后再继续
3. **全量生成**：按集批量生成，每集生成完成后自动校验
4. **自动分类**：生成的图片按集数、单元、镜号自动命名归档
5. **二次校验**：全量生成完成后进行全量人工抽检，抽检率≥30%

#### 3. 命名规范
```
S01E[集数]-U[单元号]-J[镜号]-[时长].png
例：S01E01-U01-J01-0-3s.png
```

---

### 四、问题修正机制
#### 1. 问题分类与处理流程
| 问题类型 | 处理方式 | 优先级 |
|----------|----------|--------|
| 人物设定错误 | 重新生成，修正提示词中角色特征 | 最高 |
| 场景元素缺失 | 补充场景描述，重新生成 | 高 |
| 动作/表情不符 | 调整动作描述，重新生成 | 高 |
| 视觉锤不足 | 补充圆滚滚元素描述，重新生成 | 中 |
| 色彩/光影问题 | 调整风格参数，重新生成 | 中 |
|  minor瑕疵 | 后期修图处理，不影响整体则通过 | 低 |

#### 2. 修正跟踪
- 每个问题建立单独跟踪记录
- 记录问题原因、修正方案、修正结果
- 定期汇总问题，优化提示词模板
- 问题解决率要求100%

---

### 五、效率优化方案
1. **提示词模板化**：建立标准化提示词模板，减少重复输入
2. **素材复用**：常用场景、角色参数建立素材库，直接调用
3. **批量调度**：错峰调度算力资源，提高生成效率
4. **自动化校验**：开发AI自动校验工具，减少人工工作量
5. **问题库积累**：建立常见问题解决方案库，提高修正效率

---

### 六、版本管理
- 生图成果按版本号归档，V1.0为正式交付版
- 每次修正后版本号+0.1
- 保留所有历史版本，便于回溯
- 最终交付版需通过双重校验
