## 🚀 三种整合方案（选择一种）

### 方案A：最小整合（推荐新手，1小时）

**适合**：只想快速让评审团与现有1-2个节点协同工作

**步骤**：
1. 保持现有文件结构不变
2. 创建一个简单的调度文档
3. 定义2-3个常用的节点组合
4. 通过主编排器调用

**优点**：
- 快速上手
- 风险低
- 不影响现有系统

**缺点**：
- 功能有限
- 不够系统化

### 方案B：标准整合（推荐大多数人，半天）

**适合**：想要建立完整的工作流系统，但不想大改现有结构

**步骤**：
1. 创建标准目录结构（workflows/, agents/, skills/）
2. 创建节点注册表
3. 定义5-10个常用工作流
4. 配置主编排器
5. 逐步迁移节点（保留原文件）

**优点**：
- 系统化管理
- 易于扩展
- 保留原有结构

**缺点**：
- 需要一定时间
- 需要学习新概念

### 方案C：完全重构（推荐高级用户，1-2天）

**适合**：想要建立企业级工作流系统，愿意重新组织文件结构

**步骤**：
1. 重新设计目录结构
2. 迁移所有节点到统一位置
3. 创建完整的注册系统
4. 定义所有可能的工作流
5. 配置高级功能（版本管理、监控、日志）

**优点**：
- 最系统化
- 最易维护
- 功能最强大

**缺点**：
- 耗时较长
- 需要重新组织文件

### 第4步：定义常用工作流（20分钟）

创建文件：`workflows/common-workflows.yaml`

```yaml
workflows:
  # 工作流1：单独评审
  - id: "WF-REVIEW-ONLY"
    name: "内容评审"
    description: "对单个作品进行完整评审"
    nodes:
      - step: 1
        node_id: "NODE-001"
        action: "full_review"
        input:
          content: "${user_input.content}"
          type: "${user_input.type}"
        output_var: "review_result"
    output:
      - "${review_result}"

  # 工作流2：评审+优化（假设你有优化节点）
  - id: "WF-REVIEW-OPTIMIZE"
    name: "评审并优化"
    description: "评审后根据建议优化内容"
    nodes:
      - step: 1
        node_id: "NODE-001"
        action: "review"
        input:
          content: "${user_input.content}"
        output_var: "initial_review"
      
      - step: 2
        node_id: "NODE-XXX"  # 你的优化节点ID
        action: "optimize"
        input:
          content: "${user_input.content}"
          suggestions: "${initial_review.suggestions}"
        output_var: "optimized_content"
      
      - step: 3
        node_id: "NODE-001"
        action: "review"
        input:
          content: "${optimized_content}"
        output_var: "final_review"
    
    output:
      - "${optimized_content}"
      - "${final_review}"

  # 工作流3：你的自定义工作流
  - id: "WF-CUSTOM-001"
    name: "[你的工作流名称]"
    description: "[描述]"
    nodes:
      - step: 1
        node_id: "[节点ID]"
        # ... 配置
    output:
      - "[输出]"
```

## 🎯 实际使用示例

### 场景1：快速评审

**用户**：
```
评审这个短视频脚本：
[脚本内容]
```

**系统处理**：
```
主编排器识别：需要调用 NODE-001
↓
调用 content-review-agent
↓
加载 全域审美与创作专业评审团.md
↓
执行完整评审
↓
返回评审报告
```

### 场景2：评审+优化循环

**用户**：
```
@编排器 执行工作流 WF-REVIEW-OPTIMIZE
内容：[脚本内容]
```

**系统处理**：
```
步骤1：评审原始内容
↓
步骤2：根据建议优化
↓
步骤3：评审优化后的内容
↓
返回：优化版本 + 最终评审报告
```

### 场景3：复杂自定义流程

**用户**：
```
我需要：
1. 先评审我的脚本
2. 如果评分低于120，就优化
3. 再次评审
4. 重复直到达到120分或迭代3次
```

**系统处理**：
```
主编排器动态构建工作流：
循环开始
  ├─ 评审（NODE-001）
  ├─ 判断分数
  ├─ 如果<120 → 优化（NODE-XXX）
  └─ 迭代计数
循环结束
返回最终结果
```

## 🔧 常见问题

### Q1：我的节点文件很多，都要移动吗？
**A**：不需要！保持原位置，只需在注册表中记录路径即可。

### Q2：如何添加新节点？
**A**：
1. 在 `integrations/node-registry.json` 中添加节点信息
2. 如果需要Agent，在 `agents/agent-registry.json` 中添加
3. 重启编排器或执行 `@编排器 刷新注册表`

### Q3：工作流可以嵌套吗？
**A**：可以！一个工作流可以调用另一个工作流作为子步骤。

### Q4：如何调试工作流？
**A**：
```
@编排器 执行工作流 WF-XXX --debug
```
会显示每一步的详细执行信息。

### Q5：可以暂停/恢复工作流吗？
**A**：目前不支持，但可以设置断点：
```yaml
- step: 3
  node_id: "NODE-XXX"
  breakpoint: true  # 执行到这里会暂停，等待确认
```
