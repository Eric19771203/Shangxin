## 🚀 核心功能

### 1. 智能路由
主编排器能够：
- 自动识别用户意图
- 匹配最合适的节点/工作流
- 动态组合多个节点
- 处理复杂的嵌套调用

### 2. 工作流编排
支持多种执行模式：
- **串行执行**: A → B → C
- **并行执行**: A + B + C → 整合
- **条件分支**: IF-ELSE 路由
- **循环迭代**: 优化直到达标

### 3. SubAgent委托
- 每个节点对应一个SubAgent
- SubAgent加载对应的Skill知识库
- 独立执行复杂任务
- 支持工具调用 (readFile, fsWrite, grepSearch)

### 4. 资源管理
- 统一的注册表管理
- 节点-Agent-Skill映射
- 版本控制
- 状态监控

## 💡 使用示例

### 示例1: 简单节点调用
```
用户: 评审我的短视频脚本
系统: 调用 NODE-001 (内容评审节点)
     委托 content-review-agent 执行
     返回评审报告
```

### 示例2: 预定义工作流
```
用户: @编排器 执行工作流 WF-002
     主题: 美食探店
     风格: 轻松幽默
系统: 执行 创意到剧本 工作流
     步骤1: NODE-002 生成创意
     步骤2: NODE-005 创作剧本
     步骤3: NODE-001 评审剧本
     返回完整结果
```

### 示例3: 动态组合
```
用户: 生成3个创意，评审后选最好的，然后创作剧本
系统: 动态构建工作流
     并行: NODE-002 × 3
     并行: NODE-001 × 3
     选择: 最高分创意
     串行: NODE-005 创作剧本
     返回最终剧本
```

## 📞 如何使用

### 方式1: 自然语言 (推荐)
直接描述需求，主编排器自动识别并执行
```
"评审我的脚本"
"创作一个爽文短剧"
"把这个小说改编成短剧"
```

### 方式2: 命令模式
使用 @编排器 前缀明确指定操作
```
@编排器 执行节点 NODE-001
@编排器 执行工作流 WF-002
@编排器 列出所有节点
```

### 方式3: 工作流模式
对于复杂任务使用预定义工作流
```
@编排器 执行工作流 WF-004
输入: [网文内容]
参数: [配置]
```
