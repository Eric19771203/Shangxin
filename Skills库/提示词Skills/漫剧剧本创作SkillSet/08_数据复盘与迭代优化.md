# 08_数据复盘与迭代优化 (Data Review & Iteration Skill)

## 核心功能
基于上线后的数据，自动反哺全流程优化，形成「创作→上线→复盘→优化」的全自动闭环，让内容从「靠灵感」转向「数据驱动」，大幅提升爆款概率。

## 输入内容（总控中枢自动传入）
- 成片上线后的数据：每10秒完播率、点赞率、评论率、转发率、涨粉率、观众流失节点

## 输出成果
- 数据复盘报告、剧情节奏优化方案、人设/钩子优化建议、下一集内容迭代方向、全流程参数优化指令

## 配套工具
- 数据工具：新抖、新快、平台创作者后台
- 大模型：Claude 3.5 Sonnet、GPT-4o

## 自动化执行流程
1.  自动抓取成片上线后72小时的核心数据，拆解每10秒的观众流失率，定位剧情节奏问题
2.  自动分析评论区关键词，提取观众好评点、槽点、期待点，生成内容优化方向
3.  自动将优化指令拆解到对应Skill模块，自动触发下一集内容的迭代优化（比如：30秒处流失率高，自动优化剧本冲突前置；角色好评率高，自动加戏强化人设）
4.  自动更新全流程的爆款标准库，持续优化总控中枢的校验规则，越用越贴合爆款逻辑

## 核心落地提示词（直接复制）
```markdown
你是漫剧数据运营专家，基于我输入的漫剧上线数据，生成专业的复盘报告与迭代优化方案。
要求：
1. 拆解核心数据：前3秒完播率、每10秒完播率、整体完播率、点赞率、评论率、转发率、涨粉率，定位观众流失核心节点
2. 分析评论区关键词，提取观众好评点、槽点、期待点，明确内容优化方向
3. 输出针对性的优化方案，拆解到对应环节：开篇钩子优化、剧情节奏调整、人设强化、结尾钩子升级
4. 自动生成下一集的迭代创作指令，直接传入剧本创作环节，实现全自动闭环优化
```

## 行业验证核心优化逻辑
- 核心指标：前3秒完播率＜70%，直接优化开篇钩子；单集整体完播率＜30%，优化剧情节奏和结尾钩子
- 评论关键词正向占比＞60%的人设、爽点，后续内容持续放大；负向占比高的内容，自动调整优化
