## 一、提示词架构总览

```
                    ┌──────────────────────────────────┐
                    │       PromptI18n Service          │
                    │  (application/services/           │
                    │   prompt_i18n.go)                 │
                    └───────────────┬──────────────────┘
                                    │
        ┌───────────────┬───────────┼───────────┬───────────────┐
        │               │           │           │               │
        ▼               ▼           ▼           ▼               ▼
  ┌───────────┐  ┌───────────┐ ┌────────┐ ┌────────┐    ┌───────────┐
  │ 剧本创作  │  │ 内容提取  │ │ 分镜头 │ │ 帧生成 │    │ 视频生成  │
  │ 大纲/剧本 │  │ 角色/场景 │ │ 拆解   │ │ 首/关键│    │ 约束指令  │
  │           │  │ /道具    │ │        │ │ /尾帧  │    │           │
  └───────────┘  └───────────┘ └────────┘ └────────┘    └───────────┘
        │               │           │           │               │
        └───────────────┴───────────┼───────────┴───────────────┘
                                    │
                            ┌───────▼───────┐
                            │  风格系统      │
                            │  (9种视觉风格) │
                            └───────────────┘
```

**关键设计原则**：
- **集中管理**：所有提示词集中在 `prompt_i18n.go` 一个文件中，便于维护和审计
- **双语支持**：每个提示词都有中文和英文版本，通过 `IsEnglish()` 动态选择
- **参数化**：通过 `style`、`imageRatio` 等参数注入，实现提示词的动态组装
- **严格输出约束**：所有提示词均要求 AI 返回纯 JSON 格式，避免 markdown 包裹

## 二、剧本创作类提示词

### 2.1 大纲生成提示词

| 属性 | 值 |
|------|-----|
| **函数名** | `GetOutlineGenerationPrompt()` |
| **源码行号** | `prompt_i18n.go:404-445` |
| **调用方** | `ScriptGenerationService` |
| **AI 角色** | 专业短剧编剧 |
| **输入参数** | 无 |
| **动态变量** | 主题、类型偏好、风格要求、集数（通过 FormatUserPrompt 注入） |

**用途**：根据用户提供的主题和集数，生成完整的短剧大纲，规划每集的剧情走向。

**中文提示词**：
```
你是专业短剧编剧。根据主题和剧集数量，创作完整的短剧大纲，规划好每一集的剧情走向。

要求：
1. 剧情紧凑，矛盾冲突强烈，节奏快
2. 每集都有独立的矛盾冲突，同时推进主线
3. 角色弧光清晰，成长变化明显
4. 悬念设置合理，吸引观众继续观看
5. 主题明确，情感内核清晰

输出格式：
返回一个JSON对象，包含：
- title: 剧名（富有创意和吸引力）
- episodes: 分集列表，每集包含：
  - episode_number: 集数
  - title: 本集标题
  - summary: 本集内容概要（50-100字）
  - conflict: 主要矛盾点
  - cliffhanger: 悬念结尾（如有）
```

**设计说明**：
- 强调"紧凑"和"快节奏"，因为短剧时长有限（每集约3分钟）
- 要求每集有独立冲突又推进主线，确保剧情的连贯性和单集可看性
- 悬念设置是短剧留存率的关键

### 2.2 分集剧本生成提示词

| 属性 | 值 |
|------|-----|
| **函数名** | `GetEpisodeScriptPrompt()` |
| **源码行号** | `prompt_i18n.go:544-589` |
| **调用方** | `ScriptGenerationService` |
| **AI 角色** | 专业短剧编剧 |
| **输入参数** | 无 |
| **动态变量** | 大纲内容、角色设定、集数（通过 FormatUserPrompt 注入） |

**用途**：将大纲中的概要扩展为每集 800-1200 字的详细剧本。

**中文提示词**：
```
你是一个专业的短剧编剧。你擅长根据分集规划创作详细的剧情内容。

你的任务是根据大纲中的分集规划，将每一集的概要扩展为详细的剧情叙述。
每集约180秒（3分钟），需要充实的内容。

要求：
1. 将大纲中的概要扩展为具体的剧情发展
2. 写出角色的对话和动作，不是简单描述
3. 突出冲突的递进和情感的变化
4. 增加场景转换和氛围描写
5. 控制节奏，高潮在2/3处，结尾有收束
6. 每集800-1200字，对话丰富
7. 与角色设定保持一致

输出格式：
**重要：必须只返回纯JSON对象，不要包含任何markdown代码块、说明文字或其他内容。
直接以 { 开头，以 } 结尾。**

- episodes: 分集列表，每集包含：
  - episode_number: 集数
  - title: 本集标题
  - script_content: 详细剧本内容（800-1200字）
```

**设计说明**：
- 每集约 180 秒 = 800-1200 字，这是经过计算的合理对应
- "高潮在2/3处"遵循了经典三幕式戏剧结构
- 强调"对话和动作"而非"描述"，因为后续需要拆解为可视化的分镜头

## 三、内容提取类提示词

### 3.1 角色提取提示词

| 属性 | 值 |
|------|-----|
| **函数名** | `GetCharacterExtractionPrompt(style)` |
| **源码行号** | `prompt_i18n.go:447-491` |
| **调用方** | `ScriptGenerationService.GenerateCharacters()` |
| **AI 角色** | 专业角色分析师 |
| **输入参数** | `style` - 画风风格 |
| **图片比例** | 16:9 |

**用途**：从剧本中自动提取所有有名字的角色，生成详细的角色设定卡。

**中文提示词**：
```
你是一个专业的角色分析师，擅长从剧本中提取和分析角色信息。

你的任务是根据提供的剧本内容，提取并整理剧中出现的所有角色的详细设定。

要求：
1. 提取所有有名字的角色（忽略无名路人或背景角色）
2. 对每个角色，提取以下信息：
   - name: 角色名字
   - role: 角色类型（main/supporting/minor）
   - appearance: 外貌描述（150-300字）
   - personality: 性格特点（100-200字）
   - description: 背景故事和角色关系（100-200字）
3. 外貌描述要足够详细，适合AI生成图片，包括：性别、年龄、体型、面部特征、
   发型、服装风格等,但不要包含任何场景、背景、环境等信息
4. 主要角色需要更详细的描述，次要角色可以简化
- **风格要求**：{style}
- **图片比例**：16:9

输出格式：
**重要：必须只返回纯JSON数组，不要包含任何markdown代码块、说明文字或其他内容。
直接以 [ 开头，以 ] 结尾。**
每个元素是一个角色对象，包含上述字段。
```

**设计说明**：
- 外貌描述字数要求 150-300 字，这是为后续 AI 图片生成提供足够的细节
- 明确排除"场景、背景、环境"信息，避免污染角色图片生成时的提示词
- 风格参数注入，确保角色描述与选定画风一致

### 3.2 场景提取提示词

| 属性 | 值 |
|------|-----|
| **函数名** | `GetSceneExtractionPrompt(style)` |
| **源码行号** | `prompt_i18n.go:146-197` |
| **调用方** | `ImageGenerationService.extractBackgroundsFromScript()` |
| **AI 角色** | 场景分析专家 |
| **输入参数** | `style` - 画风风格 |
| **图片比例** | 16:9 |

**用途**：从剧本中提取所有唯一的场景背景，生成纯背景（无人物）的图片提示词。

**中文提示词**：
```
【任务】从剧本中提取所有唯一的场景背景

【要求】
1. 识别剧本中所有不同的场景（地点+时间组合）
2. 为每个场景生成详细的**中文**图片生成提示词（Prompt）
3. **重要**：场景描述必须是**纯背景**，不能包含人物、角色、动作等元素
4. Prompt要求：
   - **必须使用中文**，不能包含英文字符
   - 详细描述场景、时间、氛围、风格
   - 必须明确说明"无人物、无角色、空场景"
   - 要符合剧本的题材和氛围
   - **风格要求**：{style}
   - **图片比例**：16:9

【输出格式】
**重要：必须只返回纯JSON数组，不要包含任何markdown代码块、说明文字或其他内容。
直接以 [ 开头，以 ] 结尾。**

每个元素包含：
- location：地点（如"豪华办公室"）
- time：时间（如"下午"）
- prompt：完整的中文图片生成提示词（纯背景，明确说明无人物）
```

**设计说明**：
- 强调"纯背景"和"无人物"是关键约束，因为场景图后续会作为背景与角色图合成
- 场景以"地点+时间"组合去重，避免重复生成
- 中文模式下要求中文提示词，英文模式下要求英文提示词

## 四、分镜头生成提示词

### 4.1 分镜系统提示词

| 属性 | 值 |
|------|-----|
| **函数名** | `GetStoryboardSystemPrompt()` |
| **源码行号** | `prompt_i18n.go:33-143` |
| **调用方** | `StoryboardService.GenerateStoryboard()` |
| **AI 角色** | 资深影视分镜师（精通罗伯特·麦基理论） |
| **输入参数** | 无 |
| **辅助数据** | 角色列表、场景列表（通过 FormatUserPrompt 注入） |

**用途**：将剧本按"独立动作单元"拆解为详细的分镜头方案，这是整个工作流的核心环节。

**中文提示词**：
```
【角色】你是一位资深影视分镜师，精通罗伯特·麦基的镜头拆解理论，擅长构建情绪节奏。

【任务】将小说剧本按**独立动作单元**拆解为分镜头方案。

【分镜拆解原则】
1. **动作单元划分**：每个镜头必须对应一个完整且独立的动作
   - 一个动作 = 一个镜头（角色站起来、走过去、说一句话、做一个反应表情等）
   - 禁止合并多个动作（站起+走过去应拆分为2个镜头）

2. **景别标准**（根据叙事需要选择）：
   - 大远景：环境、氛围营造
   - 远景：全身动作、空间关系
   - 中景：交互对话、情感交流
   - 近景：细节展示、情绪表达
   - 特写：关键道具、强烈情绪

3. **运镜要求**：
   - 固定镜头：稳定聚焦于一个主体
   - 推镜：接近主体，增强紧张感
   - 拉镜：扩大视野，交代环境
   - 摇镜：水平移动摄像机，空间转换
   - 跟镜：跟随主体移动
   - 移镜：摄像机与主体同向移动

4. **情绪与强度标记**：
   - emotion：简短描述（兴奋、悲伤、紧张、愉快等）
   - emotion_intensity：用箭头表示情绪等级
     * 极强 ↑↑↑ (3)：情绪高峰、高度紧张
     * 强 ↑↑ (2)：情绪明显波动
     * 中 ↑ (1)：情绪有所变化
     * 平稳 → (0)：情绪不变
     * 弱 ↓ (-1)：情绪回落

【输出要求】
1. 生成一个数组，每个元素是一个镜头，包含：
   - shot_number：镜头号
   - scene_description：场景（地点+时间，如"卧室内，早晨"）
   - shot_type：景别（大远景/远景/中景/近景/特写）
   - camera_angle：机位角度（平视/仰视/俯视/侧面/背面）
   - camera_movement：运镜方式（固定/推镜/拉镜/摇镜/跟镜/移镜）
   - action：动作描述
   - result：动作完成后的画面结果
   - dialogue：角色对话或旁白（如有）
   - emotion：当前情绪
   - emotion_intensity：情绪强度等级（3/2/1/0/-1）

**重要：必须只返回纯JSON数组，不要包含任何markdown代码块、说明文字或其他内容。
直接以 [ 开头，以 ] 结尾。**

【重要提示】
- 镜头数量必须与剧本中的独立动作数量匹配（不允许合并或减少）
- 每个镜头必须有明确的动作和结果
- 景别选择必须符合叙事节奏（不要连续使用同一景别）
- 情绪强度必须准确反映剧本氛围变化
```

**设计说明**：
- 这是项目中最复杂、最核心的提示词
- 引用"罗伯特·麦基"增强 AI 对电影叙事理论的理解
- "独立动作单元"原则确保每个镜头对应一个可生成的画面
- 情绪强度标记（-1 到 3）为后续视频生成的动态程度提供参考
- `action` + `result` 的设计分离了"过程"和"结果"，前者用于视频生成，后者用于图片生成
- 在实际调用时，系统会注入可用角色列表和场景列表，AI 需从中选择匹配的 ID

## 五、帧图片生成类提示词

这一类提示词负责为每个分镜头生成具体的 AI 图片提示词。根据帧类型的不同，有四种变体。

### 5.1 首帧提示词

| 属性 | 值 |
|------|-----|
| **函数名** | `GetFirstFramePrompt(style)` |
| **源码行号** | `prompt_i18n.go:199-237` |
| **调用方** | `FramePromptService` |
| **AI 角色** | 专业图像生成提示词专家 |
| **帧类型** | `first` - 首帧 |

**用途**：生成镜头的首帧图片提示词 — 动作发生之前的静态初始状态。

**中文提示词**：
```
你是一个专业的图像生成提示词专家。请根据提供的镜头信息，生成适合用于AI图像生成的提示词。

### 6.1 通用视频约束提示词

| 属性 | 值 |
|------|-----|
| **函数名** | `GetVideoConstraintPrompt(referenceMode)` |
| **源码行号** | `prompt_i18n.go:884-917` |
| **调用方** | `VideoGenerationService` |
| **适用模式** | `single`（单图）、`first_last`（首尾帧）、`multiple`（多图） |

**用途**：为视频生成模型提供物理一致性和运动逻辑的约束指令。

**中文提示词**：
```
### 角色定义

你是一个顶级的视频动态分析师与合成专家。你能够仅凭一张静态图或一组起始/结束帧，
精准识别画面中的物理属性、光影流向及潜在的运动趋势，生成符合物理定律的高质量视频。

### 核心执行逻辑

1. **模式识别：**
   * **单图模式（Single Image）：** 将输入图视为 Frame 0。分析画面中的"张力点"
     （如倾斜的身体、流动的液体、眼神的方向），并向该方向延续动作。
   * **双图模式（First & Last Frames）：** 严格锚定第一张图为起始，第二张图为终点。
     通过**语义插值算法**，计算两图之间所有元素的位移轨迹。

2. **物理一致性（Physics Preservation）：**
   * **质量守恒：** 确保物体在运动过程中体积、密度和材质质感不发生突变。
   * **运动惯性：** 遵循经典力学，起步平稳，加速自然，停止时不应有生硬的切断感。

3. **环境外推：** 自动补充主画面之外的背景延伸，确保运镜（Pan/Tilt/Zoom）时
   不会出现画面空洞或黑边。
```

**设计说明**：
- 区分"单图模式"和"双图模式"，指导视频模型不同的处理方式
- "张力点"分析是引导 AI 从静态图中推断运动方向的关键
- 物理一致性约束避免视频中出现形变、闪烁等常见问题
- 环境外推解决运镜时的画面边缘问题

- **色彩美学逻辑**：使用"素雅沉稳的中性色调"。
  低饱和、高明度的色彩（米白、石青、灰褐），配以小面积暗红色或金色作为点缀。
- **光影表现手法**：强调"电影级动态光影"。
  明亮的侧逆光在人物边缘勾勒出金边（Rim Light）。
```

**视觉特征**：PBR 渲染 | SSS 皮肤 | 中性素雅 | 侧逆光金边 | 环境遮蔽

### 7.9 3D 盲盒潮玩风格 (chibi3d)

## 九、提示词调用链路图

以下展示了从用户操作到 AI 调用的完整提示词组装流程：

### 1. 大纲生成流程

```
用户输入主题 + 集数
        │
        ▼
FormatUserPrompt("outline_request", theme)
+ FormatUserPrompt("genre_preference", genre)        ──► User Prompt
+ FormatUserPrompt("episode_count", count)
        │
        ▼
GetOutlineGenerationPrompt()                         ──► System Prompt
        │
        ▼
AIService.GenerateText(userPrompt, systemPrompt)
        │
        ▼
JSON: { title, episodes: [...] }
```

### 2. 角色提取流程

```
剧本内容 + 风格
        │
        ▼
FormatUserPrompt("character_request", script, maxCount) ──► User Prompt
        │
        ▼
GetCharacterExtractionPrompt(style)                      ──► System Prompt
        │
        ▼
AIService.GenerateText(userPrompt, systemPrompt)
        │
        ▼
JSON: [{ name, role, appearance, personality, description }, ...]
```

### 3. 分镜头生成流程

```
剧本 + 角色列表 + 场景列表
        │
        ▼
script_content_label + 剧本
+ character_list_label + 角色JSON                        ──► User Prompt
+ scene_list_label + 场景JSON
+ task_instruction
+ character_constraint
+ scene_constraint
        │
        ▼
GetStoryboardSystemPrompt()                              ──► System Prompt
        │
        ▼
AIService.GenerateText(userPrompt, systemPrompt)
        │
        ▼
JSON: [{ shot_number, scene_description, shot_type, camera_angle,
         camera_movement, action, result, dialogue, emotion,
         emotion_intensity }, ...]
```

### 4. 帧图片生成流程

```
分镜头信息 + 风格 + 帧类型
        │
        ▼
FormatUserPrompt("frame_info", shotContext)              ──► User Prompt
  包含：镜头描述、场景、角色、动作、结果、
        对白、氛围、景别、角度、运镜
        │
        ▼
根据帧类型选择系统提示词:                                ──► System Prompt
  first  → GetFirstFramePrompt(style)
  key    → GetKeyFramePrompt(style)
  last   → GetLastFramePrompt(style)
  action → GetActionSequenceFramePrompt(style)
        │
        ▼
GetStylePrompt(style)                                   ──► 风格前缀叠加
        │
        ▼
AIService.GenerateText(userPrompt, systemPrompt + stylePrompt)
        │
        ▼
JSON: { prompt: "图片生成提示词", description: "简短描述" }
```

### 5. 视频生成流程

```
图片 + 提示词 + 参考模式
        │
        ▼
用户提供的场景描述                                       ──► Prompt
        │
        ▼
GetVideoConstraintPrompt(referenceMode)                  ──► 约束提示词
  single          → 通用约束（单图模式）
  first_last      → 通用约束（双图模式）
  action_sequence → 九宫格专用约束
        │
        ▼
VideoClient.GenerateVideo(imageURL, prompt + constraint, options)
        │
        ▼
VideoResult: { videoURL, duration, ... }
```
