# 资产筛选与映射 - 提示词

| 属性 | 值 |
|------|------|
| **源文件** | `src/agents/storyboard/generateImageTool.ts` |
| **类型** | 硬编码用户消息 |
| **AI配置组** | `storyboardAgent` |
| **输出格式** | 结构化 `{ relevantAssets: [{name, reason}] }` |
| **修改方式** | 需修改源文件中的提示词字符串 |

---

## 资产筛选提示词原文

用于从项目所有可用资产中，AI 筛选出与当前分镜内容直接相关的资产：

```
请分析以下分镜描述，从可用资产中筛选出与分镜内容直接相关的资产。

分镜描述：
1. ${prompts[0]}
2. ${prompts[1]}
...

可用资产列表：
- ${resource.name}：${resource.intro}
...

请仅选择在分镜中明确出现或被提及的角色、场景、道具。不要选择与分镜内容无关的资产。
```

---

## 资产映射提示词（buildResourcesMapPrompts）

筛选后的资产会被映射为编号，用于图片生成时的系统提示词：

```
其中人物、场景、道具参考对照关系如下：${name1}=图片1, ${name2}=图片2, ...
```

### 映射规则

- 前 9 张图片：`${name}=图片${index+1}`
- 第 10 张及以后：`${name}=图10-${index-8}`（因为第10张是合并图）

---

## 图片处理规则

| 条件 | 处理方式 |
|------|----------|
| 图片数量 ≤ 10 | 每张单独压缩到 3MB 以内 |
| 图片数量 > 10 | 前 9 张单独压缩，第 10 张及之后的图片拼接合并为一张 |
| 总大小 > 10MB | 按大小降序逐一压缩，直到总大小 ≤ 10MB |

---

## 数据流

1. 从大纲（`t_outline`）获取角色/场景/道具列表
2. 从资产表（`t_assets`）查询已有图片
3. AI 筛选相关资产 → 获取 `filteredImages`
4. 构建映射提示词 → 作为图片生成的 `systemPrompt`
5. 压缩/合并图片 → 作为图片生成的 `imageBase64` 参考
