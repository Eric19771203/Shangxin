# 世界观搭建 SKILL

## 1. 简介
本 SKILL 专为生成“电影级、逻辑闭环、可落地”的完整世界观而设计，融合了专业的编剧世界观构建法则、标准化的定稿模板、多类型对标范例以及自动化的一致性校验机制。旨在帮助创作者高效、系统地构建出能够支撑长篇连载（如漫剧、网文、剧本）的坚实世界观底座。

## 2. 核心架构
本 SKILL 采用“分层构建 + 自动化校验”的架构：

*   **核心构建引擎 (Core Building Engine)**：
    *   **法则库 (Methodology)**：基于 `world-building-method.md`，提供冰山法则、冲突驱动、角色绑定等核心构建原则。
    *   **视觉库 (Visual Style)**：基于 `visual-style.md`，提供电影级质感、漫剧分镜标准的视觉规范。
    *   **模板库 (Templates)**：提供核心概念、完整世界观、角色绑定、场景描述的标准化模板。
    *   **范例库 (Examples)**：提供都市异能、东方玄幻、赛博朋克等类型的对标范例。

*   **独立质检卫士 (Quality Assurance Agent)**：
    *   **Aligner Subagent**：负责在世界观构建过程中对逻辑闭环、规则统一性、冲突驱动力等进行全方位检查。

## 3. 运行模式
本 SKILL 采用分阶段流式创作模式，用户通过指令驱动创作流程：

1.  **Stage 1: 核心概念 (Core Concept)**：确立世界名称、时代、核心规则、稀缺资源与核心矛盾。
2.  **Stage 2: 体系深化 (System Deepening)**：扩展地理、社会、文化、历史等维度，构建完整世界观。
3.  **Stage 3: 角色绑定 (Character Integration)**：将世界观规则与主角/反派的动机、困境绑定。
4.  **Stage 4: 视觉化呈现 (Visual Presentation)**：生成关键场景的画面描述、分镜概念。
5.  **一致性校验 (Alignment Check)**：Aligner Subagent 在关键节点介入，检查逻辑漏洞，输出 PASS/FAIL 及修改建议。

## 4. 角色与职责

### 4.1 架构师 Agent (Architect)
*   **职责**：负责根据用户输入和当前阶段，调用相应的法则、模板和范例，构建高质量的世界观内容。
*   **能力**：
    *   熟练掌握各类题材（奇幻/科幻/悬疑）的世界观构建法则。
    *   能够运用视觉语言描述抽象概念。
    *   具备系统化思维，能够确保世界观的逻辑自洽。

### 4.2 质检 Agent (Aligner)
*   **职责**：负责对架构师 Agent 生成的内容进行严格的一致性校验。
*   **能力**：
    *   精通逻辑闭环验证、冲突驱动验证、角色绑定验证。
    *   能够识别规则冲突、动力不足和设定脱节等问题。
    *   严格执行检查标准，提供具体的修改建议。

## 5. 核心文件清单
*   `SKILL.md`: 核心配置文件，定义元数据、能力和接口。
*   `world-building-method.md`: 世界观构建法则。
*   `visual-style.md`: 视觉规范指南。
*   `templates/`: 标准化模板文件夹。
*   `examples/`: 对标范例文件夹。
*   `world-aligner-subagent/`: 质检 Agent 提示词和描述文件。

## 6. 使用指南
1.  **初始化**：用户输入一句话灵感或题材类型，启动核心概念生成 (`/start`)。
2.  **分阶段执行**：按顺序执行 Deepen -> Bind -> Visualize 阶段。
3.  **质检反馈**：关注 Aligner 的反馈，根据建议调整设定。
4.  **迭代优化**：通过多轮对话微调世界观细节，直至满意。
