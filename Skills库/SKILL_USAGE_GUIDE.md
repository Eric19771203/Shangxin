# Skill使用指南

## 一、技能库概述

欢迎使用Skills库！这是一个专注于剧本写作与漫剧生成的专业工具集合，包含了多种创作技能，可以帮助你完成从故事构思到成品生成的全流程创作。

### 核心功能
- **小说创作**：古代权谋、现代言情、现实题材等多种类型
- **漫剧创作**：末世重生、玄幻、网文改编等多种题材
- **分镜设计**：电影分镜、动画分镜、分镜评审等专业工具
- **全流程支持**：从大纲、人物、目录到章节的完整创作流程

### 设计理念
- **模块化**：每个技能独立封装，可单独使用
- **标准化**：统一的调用格式和数据结构
- **可扩展**：支持自定义技能和流程
- **专业化**：基于专业创作方法论和模板

## 三、技能调用方法

### 1. 单技能调用

#### 基本格式
```markdown
[SKILL_CALL]
skill_id: SKILL-XXX
mode: full|partial
input:
  stage: "创作阶段"
  content: "创作内容或需求"
  config:
    style: "风格配置"
    length: "长度配置"
output_to: "输出文件路径"
[/SKILL_CALL]
```

#### 示例：生成小说大纲
```markdown
[SKILL_CALL]
skill_id: SKILL-001
mode: full
input:
  stage: "outline"
  content: "创作一个关于宫廷权谋的故事大纲"
  config:
    style: "文白相济"
    length: "medium"
output_to: outline.md
[/SKILL_CALL]
```

#### 示例：生成漫剧人物
```markdown
[SKILL_CALL]
skill_id: SKILL-007
mode: full
input:
  stage: "character"
  content: "为末世重生漫剧创作主要人物"
  config:
    style: "紧张刺激"
    length: "detailed"
output_to: character.md
[/SKILL_CALL]
```

### 2. 链式调用

#### 基本格式
```markdown
[SKILL_CHAIN]
chain_name: "创作全流程"

step_1:
  skill_id: SKILL-XXX
  action: "动作描述"
  input:
    stage: "创作阶段"
    content: "创作内容"
  output_var: "变量名"

step_2:
  skill_id: SKILL-XXX
  action: "动作描述"
  input_from: "变量名"
  output_var: "变量名"

# 更多步骤...
[/SKILL_CHAIN]
```

#### 示例：小说创作全流程
```markdown
[SKILL_CHAIN]
chain_name: "古代权谋小说创作全流程"

step_1:
  skill_id: SKILL-001
  action: "生成大纲"
  input:

  output_var: "outline"

step_2:
  skill_id: SKILL-001
  action: "生成人物"
  input_from: "outline"
  output_var: "characters"

step_3:
  skill_id: SKILL-001
  action: "生成目录"
  input_from: "characters"
  output_var: "catalog"

step_4:
  skill_id: SKILL-001
  action: "生成章节"
  input_from: "catalog"
  output_var: "chapters"

  skill_id: SKILL-004
  action: "生成分镜"
  input_from: "chapters"
  output_var: "storyboard"

[/SKILL_CHAIN]
```

## 五、最佳实践

### 1. 明确需求
- **具体描述**：提供详细的创作需求，包括题材、风格、长度等
- **参考示例**：如果有参考作品，提供相关信息
- **目标明确**：明确创作的最终目标和用途

### 2. 合理规划
- **分阶段创作**：按照大纲→人物→目录→章节的顺序进行
- **循序渐进**：每个阶段完成后，检查结果再进入下一阶段
- **保持一致性**：确保前后内容风格一致，逻辑连贯

### 3. 有效配置
- **风格选择**：根据题材选择合适的写作风格
- **长度控制**：根据需求设置合理的内容长度
- **格式规范**：使用标准的文件格式和命名规范

### 4. 质量控制
- **内容检查**：定期检查创作内容的质量和一致性
- **风格统一**：确保整个作品风格统一
- **逻辑连贯**：检查前后剧情的逻辑连贯性
- **细节完善**：关注人物塑造、场景描写等细节

### 5. 高效使用
- **链式调用**：对于复杂流程，使用链式调用提高效率
- **批量处理**：对于相似内容，批量生成后再修改
- **模板使用**：充分利用技能提供的模板和示例
- **迭代优化**：根据反馈不断优化创作内容

## 六、故障排除

### 1. 常见错误

| 错误码 | 错误类型 | 可能原因 | 解决方案 |
|--------|----------|----------|----------|
| 400 | 参数错误 | 输入参数不完整或格式错误 | 检查输入参数是否完整，格式是否正确 |
| 404 | 技能不存在 | 技能ID错误或技能未安装 | 检查技能ID是否正确，确保技能已安装 |
| 500 | 执行失败 | 创作过程中出现错误 | 检查输入内容是否合理，尝试简化需求 |
| 503 | 技能不可用 | 技能正在维护或暂时不可用 | 稍后再试，或联系管理员 |

### 2. 常见问题

#### 问题：生成的内容不符合预期
**解决方案**：
- 提供更详细的创作需求
- 明确指定风格和长度
- 参考示例文件调整输入

#### 问题：技能执行速度慢
**解决方案**：
- 简化创作需求
- 减少内容长度
- 分批次执行复杂任务

#### 问题：生成的内容风格不一致
**解决方案**：
- 在配置中明确指定风格
- 参考示例文件确保风格一致
- 使用链式调用保持风格统一

#### 问题：技能执行失败
**解决方案**：
- 检查输入参数是否正确
- 确保网络连接正常
- 尝试重新执行技能

### 3. 性能优化

- **合理使用缓存**：对于重复使用的内容，使用缓存提高效率
- **批量处理**：对于相似内容，批量生成后再修改
- **优化输入**：提供清晰、简洁的输入内容
- **使用异步调用**：对于长时间运行的任务，使用异步调用

## 七、版本管理

### 版本号格式
`MAJOR.MINOR.PATCH`
- **MAJOR**：重大变更，不兼容之前版本
- **MINOR**：新增功能，兼容之前版本
- **PATCH**：修复bug，兼容之前版本

### 版本更新
- 定期检查技能版本更新
- 查看更新日志了解新功能
- 测试新版本是否符合需求
- 按需更新技能版本

### 版本兼容
- 不同版本的技能可能有不同的参数格式
- 链式调用时，确保使用兼容的技能版本
- 参考技能文档了解版本差异
